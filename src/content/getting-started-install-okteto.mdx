This document outlines the steps to set up your cluster to use with Okteto.

:::info
If you don't have a Kubernetes cluster available for installing Okteto, you can try it [on our free SaaS playground](https://cloud.okteto.com).
:::

## Requirements
In order to fully install Okteto, you'll need:
1. [An Okteto License](https://www.okteto.com/free-trial/)
1. Admin access to a Kubernetes cluster
1. The Helm CLI

## Add the Okteto Helm repository

Add the Okteto Helm repository to helmâ€™s configuration:

```
helm repo add okteto https://charts.okteto.com
helm repo update
```

##  Installing Okteto

Okteto is distributed as a Helm chart. To install the chart, run the install command with the `--set` argument:

```
helm install okteto okteto/okteto --namespace=okteto --create-namespace --set license="<YOUR_LICENSE_KEY>"
```

After a few seconds, all the resources will be created. 

## Accesing your Okteto instance

By default, Okteto is not configured to accept traffic from outside the cluster on which it is installed.  

Run the command below to start a port-forward in order to be able to access your Okteto instance.

```
 $ sudo kubectl port-forward service/okteto-ingress-nginx-controller 443:443 --namespace okteto
```  

To access your Okteto instance, click on the URL provided in the CLI output. It will look like this: 

```
Access your Okteto instance at https://okteto.localtest.me/login#token=XXXXXX
```

> **Important**: The default installation is not recommended for production use. We highly advise following the steps outlined in the [Admin Dashboard](administration/dashboard.mdx) before giving your team access to your Okteto instance.
